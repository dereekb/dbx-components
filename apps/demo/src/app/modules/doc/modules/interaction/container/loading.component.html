dbx-loading is typically used in conjunction with a LoadingContext, which streams LoadingContextEvents signifying the current loading state.

There are two presentations:

Default (w/ {{ diameter }}px diameter)

The diameter can be customized on the component.

Linear

Text

Both presentations can have custom text to show while loading.

Default

Linear

Padding

The loading bar can be configured to provide padding to prevent content from jumping around when loading begins/ends. Useful in cases where the loading bar doesn't control any content but removing it would push the content upward to fill the empty space.

Custom content not pushed by the bar.

Error Example

If the loading context encounters an error, the error will be displayed.

Custom Loading Content

Custom loading content can be used to replace the other loading information by added loading as an attribute.

This is custom loading content in a pit. We're currently loading!

Custom Error Content

Custom error content can be used to replace the default dbx-error widget by added error as an attribute.

This is custom error content in a pit for the error. We encountered an error!

Custom Error Action

An error action view can be added by adding errorAction as an attribute and it will be shown when an error occurs.
